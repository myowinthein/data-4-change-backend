# Setup

This is the verified setup procedure. Every step has been tested and confirmed working.

## Prerequisites

| Tool | Required version | Notes |
|---|---|---|
| Docker Desktop | Any recent version | Must be running |
| Docker Compose | v2+ | Included with Docker Desktop |
| Node.js | **12.x** | For frontend only. Node 14 also works. Node 16+ breaks `node-sass`. |
| Yarn | Any | For frontend only. `npm` also works. |

> **Do not use the host PHP or MySQL.** The backend runs entirely inside Docker containers.

---

## Backend Setup

### 1. Clone the repository

```bash
git clone git@gitlab.com:hackathon7814206/data-for-change/backend.git
cd backend
```

### 2. Create the environment file

```bash
cp .env.example .env
```

The `.env.example` has safe placeholder values. The `.env` file is pre-configured for Docker in this repo — if you copied from example, update the DB values:

```
DB_HOST=db
DB_DATABASE=data4change
DB_USERNAME=data4change
DB_PASSWORD=secret
APP_URL=http://localhost:8100
```

> The APP_KEY will be auto-generated by the entrypoint script on first boot.

### 3. Build the Docker image

```bash
docker compose build
```

This step installs all PHP dependencies inside the image. It takes 1–3 minutes on first build. Subsequent builds are cached.

> If you see a `vendor/` directory on the host after a previous run, delete it before rebuilding:
> `rm -rf vendor/`

### 4. Start the stack

```bash
docker compose up -d
```

The startup sequence is automatic:
1. MySQL starts and imports `storage/backup/data4change_2019-05-25.sql` (~30 seconds on first run)
2. App waits for MySQL to be healthy
3. App copies vendor from image cache, generates APP_KEY, runs migrations (no-op — dump handles schema), creates storage symlink
4. Apache starts

### 5. Verify

```bash
curl http://localhost:8100/api/region/getAll?lang=en
```

Expected: JSON with 18 Myanmar regions.

Open http://localhost:8100/login in a browser.

## Frontend Setup

The frontend is a **separate repository** and is not managed by the backend `docker-compose.yml`.

```bash
# From the parent directory
cd ../frontend

# Install dependencies (first time only)
yarn install

# Start dev server
yarn dev
```

Frontend will be available at **http://localhost:3000**.

### Required: Frontend .env

The frontend repo must have a `.env` file:

```
baseURL=http://localhost:8100/api
```

This file should already be present. If not, create it.

> Node 12 or 14 is required. With Node 16+, `node-sass@4` will fail to compile.
> Use `nvm use 12` or `nvm use 14` if you have nvm installed.

## Useful Commands

```bash
# Start everything (backend only — start frontend separately)
docker compose up -d

# Stop all backend containers (data is preserved in backend_db_data volume)
docker compose down

# View live logs
docker compose logs -f app
docker compose logs -f db

# Open a shell inside the PHP container
docker exec -it backend-app-1 bash

# Connect to MySQL directly
mysql -h 127.0.0.1 -P 3307 -u data4change -psecret data4change

# Rebuild the image after Dockerfile changes
docker compose build --no-cache
rm -rf vendor/   # always clear stale vendor before rebuilding
docker compose up -d
```

## Resetting the Database

If you need a completely fresh database:

```bash
docker compose down
docker volume rm backend_db_data
docker compose up -d
```

MySQL will re-import the SQL dump automatically on next start.

> Do NOT run `php artisan migrate:refresh --seed` — the seeder truncates the users table and the SQL dump is the only source of all statistical data.
